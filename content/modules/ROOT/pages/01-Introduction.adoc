# **__Welcome to rhis-builder course - Part I__**

## Introduction

This course is part 1 of an extended course designed for Red Hat Infrastructure Standard (RHIS) implementers using the RHIS Adoption Model process with the rhis-provisioner container to deploy a Red Hat Enterprise Linux based hybrid multi-cloud environment using Red Hat management tooling. This course is rated as **Advanced**.

- Part 1 - Builds the RHIS environment
- Part 2 - Extends the RHIS environment and investigates advanced features

### **__Prerequisites:__**

It is expected that you have reviewed the https://github.com/parmstro/rhisam[Red Hat Infrastructure Standard Adoption Model (RHISAM)] documentation, have worked though at least one engagement using the Red Hat Navigate Process, and have strong experience with:

- Red Hat Enterprise Linux
- Red Hat Identity Management
- Red Hat Satellite 
- Red Hat Ansible Automation Platfrom 

If you have not done so, please review the material before attempting this course. 

This is NOT a course for someone new to Linux or new to Red Hat Enterprise Linux. Attempting this course prior to meeting the above prerequisites will only lead you to be frustrated and to have a poor experience. 

If you are attending this course through the Red Hat Demonstration Platform (RHDP), there no compute system prerequisites other than that your system has a supported browser for accessing web pages.

If you are attending this course as part of a BYOD workshop, the prerequistes are listed in the systems prerequisites below.

## What you will learn

This course builds on the foundation of learning developed through studying the RHISAM and Navigate Process by:

- Introducing the rhis-builder project
- Review of the rhis-builder core repositories
- Understanding the rhis-builder-inventory project
- Setting up the Provisioner node
- Customizing an rhis-builder-inventory repository for a target environment
- Building an RHIS deployment using the rhis-provisioner container
- Troubleshooting deployment issues
- Testing the RHIS deployment
- Understanding Next Steps

The rhis-builder project is a set of tooling and configuration designed to build, manage and maintain an opinionated model of a hybrid cloud Red Hat Enterprise Linux environment using infrastructure as code and gitOps with traditional VM and container workloads. You can start where you are, or you can build from scratch. 

### **__RHDP - Deployed Instances:__**

The deployed environment is built around your fictional domain - ${guid}.${_global_domain}

Within the environment you will have: 

- provisioner.${guid}.${_global_domain}    - this is the Provisioner for RHIS. It will host the rhis-provisioner container and manage the RHIS inventory
- idm1.${guid}.${_global_domain}           - this is the Identity Management Primary system. It will be responsible for Authentication, DNS, and Certificates
- satellite1.${guid}.${_global_domain}     - this is the Satellite Primary system. It will manage all content, provisioning, compliance reporting and rememditation
- aap1.${guid}.${_global_domain}           - this is the All-in-one AAP 2.6 server. It will manage all RHIS automation tasks and workflows.
- client1.${guid}.${_global_domain}        - this is a RHIS client system 1
- clientn.${guid}.${_global_domain}        - this is a RHIS client system n

For the part 1 environment, the network is flat.

- network: 192.168.140.0
- prefix: 22
- subnet mask: 255.255.252.0
- number of IP addresses: 1,024



### **__BYOD - What you will need at a minimum:__**

A plan! 

What sort of environment do you want to build? Literally, you can build a hybrid multicloud deployment.

Your best experience will be with 3 systems. They can be VMs, Baremetal or cloud instances for:
- a node to run the rhis-provisioner-9 container and manage your inventory
- a node for the identity management primary
- a node for satellite primary

That gets you off the ground, to get flying you will need:
- more baremetal nodes, or 
- a supported hypervisor environment, or 
- a supported cloud environment, or 
- all of the above

to build out the rest of your architecture...
